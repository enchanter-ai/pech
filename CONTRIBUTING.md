# Contributing to Pech

*An @enchanter-ai product.*

Thanks for your interest in contributing. Read [CLAUDE.md](CLAUDE.md) before opening your first PR — it's the binding behavioral contract for the plugin, including the non-duplication boundary with Emu (A2 owns tokens; L1 owns dollars) and the no-per-call-events rule for the enchanted-mcp bus.

## Ground rules

1. **Zero external runtime deps.** Hooks: bash + jq. Scripts: Python 3.8+ stdlib only. Rate-card.json is committed, refreshed by CI — never fetched at runtime.
2. **Named algorithms.** Every engine has an ID (`L1`..`L5`) and an academic-style name in `[Method] [Domain] [Action]` form. Derivations in `docs/science/README.md`.
3. **Managed agent tiers.** Opus = judgment (anomaly triage). Sonnet = executor (forecasting loop). Haiku = validator (schema checks). Never route a Haiku task to Opus — in Pech specifically, doing so is ironic: the cost-discipline plugin burning budget.
4. **Emu-style marketplace.** Each sub-plugin ships `.claude-plugin/plugin.json` + `{agents,commands,hooks,skills,state}/` + `README.md`.
5. **Diagrams from source-of-truth.** `docs/architecture/*.mmd` + `index.html` are regenerated by `docs/architecture/generate.py`. Do not hand-edit.
6. **Threshold + rollup events only.** The enchanted-mcp bus is not for per-tool-call emission. Breaking this rule is an automatic revert — see the `pech_publish` rate limiter in `shared/constants.sh`.
7. **Honest forecasts.** L1 always reports ±2σ confidence bands. Point estimates are banned.

## Dev setup

```bash
git clone https://github.com/enchanter-ai/pech
cd pech
bash install.sh                                  # pre-flight checks, clone to ~/.claude/plugins/
cd docs/assets && npm install                    # renderer toolchain (mmdc + puppeteer + mathjax)
cd ../../ && bash tests/run-all.sh               # smoke tests
```

## Before opening a PR

1. **Verify rate-card freshness.** `jq .effective_from shared/rate-card.json` — if > 60 days old, refresh via the nightly-CI PR flow.
2. **Run the generator.** `python docs/architecture/generate.py` — regenerates the four `.mmd` diagrams + `index.html`. Commit the result.
3. **Run the tests.** `bash tests/run-all.sh` — every sub-plugin's `tests/` dir must return zero.
4. **Verify the conduct modules are current.** Compare against upstream (`enchanter-ai/wixie`); if they've drifted, rebase before proceeding.
5. **Check the non-duplication boundary.** If your change makes Pech count tokens client-side, stop and reconsider — that's Emu's A2.
6. **Check the bus-traffic invariant.** `grep -r 'pech_publish' plugins/` — every caller must be inside a threshold-crossing or rollup-boundary code path. Per-call callers fail review.

## Commit conventions

Conventional Commits. Scope by sub-plugin slug:

```
feat(cost-tracker): add L4 cache-waste measurement
fix(budget-watcher): debounce threshold event per scope-window
docs(shared): update rate-card.json after Anthropic price change
refactor(cost-query): extract common formatter
test(cost-tracker): add orphan-attribution regression case
```

Scopes: `cost-tracker`, `budget-watcher`, `rate-card-keeper`, `pech-learning`, `cost-query`, `full`, `shared`, `docs`, `tests`, or `repo` for top-level.

## Reporting issues

Open at https://github.com/enchanter-ai/pech/issues. Include: plugin version (from `.claude-plugin/marketplace.json#metadata.version`), Claude Code version, rate-card date (`jq .effective_from shared/rate-card.json`), repro steps, expected vs. actual.

## Code of conduct

Be kind. Debate ideas, not people. Pech is a cost-discipline plugin; bring that same discipline to review — if a PR claims a 20% cost reduction, verify with a before/after ledger diff. Numbers hold the line.
